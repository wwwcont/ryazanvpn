# Telegram Bot User Flows

## Main menu

- Мой доступ
- Мой баланс
- Speed
- Health
- Трафик
- История кодов
- Помощь
- Перевыпустить ключи

## `/start`

1. Регистрирует/обновляет пользователя в системе.
2. Показывает короткое приветствие.
3. Объясняет два режима:
   - **Speed** — быстрый и простой.
   - **Health** — более живучий для сложных условий.
4. Показывает главное меню.
5. Если доступ уже активен — показывает статус и баланс.
6. Если доступа нет — предлагает активировать invite code.

## Мой доступ

Показывает:
- статус доступа;
- активную ноду (если есть);
- доступные протоколы (Speed/Health);
- срок действия;
- подсказку по переключению между режимами.

## Мой баланс

Показывает текущий статус баланса в UX-форме и направляет к администратору для финансовых деталей.

## Speed (AmneziaWG)

Экран Speed содержит кнопки:
- Скачать .conf
- Показать QR
- Показать текст
- Перевыпустить Speed

## Health (Xray Reality)

Экран Health содержит:
- Получить ключ/ссылку
- Инструкция подключения
- Перевыпустить Health

## Трафик

Показывает агрегированные значения:
- за всё время;
- за 30 дней;
- общим итогом по протоколам.

## История кодов

Отдаётся в безопасном формате (без полного раскрытия invite code) и без технических идентификаторов.

## State machine

Базовые состояния:
- `idle`
- `awaiting_invite_code`
- `awaiting_admin_user_lookup`
- `awaiting_balance_adjustment`
- `awaiting_confirm_reissue`
- `awaiting_confirm_block`
- `awaiting_confirm_unblock`

Состояния хранятся централизованно в Redis state-store c TTL.
